# Findings — Erich Friedman Packing Center

> Generated by Claude Opus 4.8. The author (Tej Stead) apologizes for the AI slop.

A search for improvable entries in [Erich Friedman's Packing Center](https://erich-friedman.github.io/packing/),
focused on equal-unit-circle packings (where every claim is checkable by exact
arithmetic: pairwise center distance ≥ 2, center-to-wall clearance ≥ 1). Two
problems yielded genuine new results; several other avenues were dead ends and are
recorded so they aren't re-explored.

All packings below are independently verified by `common/verify.py`, which reads only
the coordinates and shares no code with the optimizer.

## 1. Circles in an L-tromino (`cirinl`) — 24 new records, n = 17–40

The L-tromino of size `s` is the square `[0,s]²` with the top-right quarter
`[s/2,s]²` removed; `s` is the outer side. The published page stops at **n = 16**.
The optimizer (force relaxation + SLSQP, validated against proven optima elsewhere)
produces verified packings for **n = 17 through 40**. Each strictly beats the best
trivial square/hexagonal grid except n = 27, which ties it at exactly 12.

All values increase monotonically and exceed the published n = 16 record (9.635).
Coordinates: `cirinl_circles_in_L/data/packings.json`. The container is not on
Packomania, so these are first records for n ≥ 17.

Honesty note: for n = 33, 34, 38 different random seeds found nearby competing local
minima (differing by ~1e-3 to 1e-2); the smaller verified value is reported and these
are the likeliest to tighten with more search.

### No closed forms

A few of these packings (n = 23, 25, 26) initially looked rigid by a crude
degree-of-freedom count, with n = 26 even mirror-symmetric across the L's diagonal,
so they seemed candidates for exact algebraic side lengths. A high-precision analysis
refutes this. Re-solving each contact system to ~60 digits and examining the contact
Jacobian:

- the Jacobian is **rank-deficient** (n = 26: 52 contacts, rank 50, null dim **3** —
  confirmed real, with the smallest singular values ~5e-61 below a clear gap from 0.24);
- the null space has a **nonzero `s`-component** (n = 26: components 0.006–0.024, far
  above the 1e-12 noise floor).

So the contacts admit infinitesimal motions that change `s` while preserving every
contact — the packings are **"floppy in `s`."** The side length is fixed by the
inequality-constrained minimization hitting a boundary, not by a determined algebraic
system, which is exactly why no simple radical fits (PSLQ finds no stable low-degree
minimal polynomial at 60 digits). Even the symmetric n = 26 is genuinely non-rigid —
in fact the *most* floppy of the three. The only exact value is the trivial n = 27 = 12.
Details: `ANALYTIC_L.md` (in the working repo).

## 2. Circles in an arbitrary triangle (`cirinttt`) — 15 improvements

Pack `n` unit circles in the triangle of smallest area, with all three vertices free.
The page tabulates only `n ∈ {2,4,7,8,11,12,13,16,17,18,19,22}` and states that for
other `n` the best known packing uses an equilateral triangle. For `n` strictly
between triangular numbers, a tailored non-equilateral triangle does better.

The baseline is the genuinely optimal equilateral disk packing from **Graham &
Lubachevsky, "Dense packings of equal disks in an equilateral triangle: from 22 to 34
and beyond," Electron. J. Combin. 2 (1995), #A1** (their `d(n)` converts to equilateral
side `2/d(n) + 2√3`, area `(√3/4)s²`). For n = 38, 39, 41 — which G–L does not tabulate
individually — a heavy independently-validated equilateral solve was used and is
labeled as such.

### Exact closed form, and a partial optimality proof, for n = 25

The n = 25 optimum is a **30-60-90 right triangle**: short leg `9 + √3`, long leg
`3 + 9√3`, hypotenuse `18 + 2√3`, **area `27 + 42√3` ≈ 99.746134** (the 25 centers form a
spacing-2 triangular lattice block). This is verified symbolically exact (min pair
distance = 2, min wall clearance = 1, area = 27 + 42√3, all exact).

An attempted optimality proof established, rigorously: feasibility (exact); an exact
objective reduction `Area = Area_inner + Perimeter_inner + Σcot(θᵢ/2)`; that the
configuration is infinitesimally rigid (jammed) and a strict local minimum; and that it
is **optimal among all triangular-lattice arrangements** of 25 points (finite enumeration
over 11,213 valid lattice triangles). A complete global proof was **not** achieved: the
only general tool, Oler's inequality (1961), is tight only for equilateral lattice
triangles and has slack exactly `2√3 − 2 ≈ 1.464` here, giving a global lower bound of
only 92.83 (7.45% short). No corner-aware inequality tight for 30-60-90 lattice triangles
is known. A literature check found no published optimality proof for this free-shape
problem at any n, and no published n = 25 value, so the result is novel/unpublished.
Details: `PROOF_n25.md` (in the working repo).

## 3. Avenues that did not yield (recorded to avoid re-exploring)

- **Fixed-container circle packings (`cirinpen`, `cirinhex`, `cirinl` n ≤ 16):** the
  optimizer reproduces every published record to its printed precision. These are
  already tight; no improvement.
- **Circles in arbitrary ellipses (`cirinel`):** a validated ellipse packer (reproduces
  Cantrell's closed forms exactly) matched but did not beat any 2007 record across
  n = 5,6,8,10–14,16–23. The table is mature; Pólik's n = 9, 15 (Jan 2026) were the only
  soft spots and are already taken.
- **Covering problems (`circovsqu`, `squcotri`):** a covering optimizer + rigorous
  coverage verifier (validated by reproducing all proved values: √2, 4/√5, 2√2, 1+√7,
  …) came up just under every published "found" record but beat none. For `squcotri` the
  smooth optimizer could not even reproduce the known n = 4 value (the records use
  specific tilted/pinwheel arrangements), so no claim was attempted. Clean negative.

## Method and validation

The optimizer (`common/solver_source/`) was validated three independent ways before any
claim: it reproduces proven-optimal circles-in-equilateral-triangle side lengths
(n = 4,5,7,9,12,15) to ~1e-13 and never returns a value below a proven optimum; it
reproduces existing `cirinpen`/`cirinhex`/`cirinl` (n ≤ 16) records to printed precision;
and every claimed packing is re-checked by the standalone `common/verify.py`. New values
were compared against the appropriate best-known baselines (grids for the L,
Graham–Lubachevsky for the triangle) and claimed only when strictly better.

These are best-known-style numerical results, not optimality proofs (except the n = 25
partial proof above).
